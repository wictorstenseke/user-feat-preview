# US-02 Feedback Composer

**Status: Done**

## Summary

An AI-style feedback composer at the top of the main page that collects raw user input and displays an editable draft generated by the LLM.

## User Story

As a visitor to the feedback portal,  
I want an AI-style input box that helps structure my feedback,  
so that I can quickly submit high-quality feature or bug reports without knowing the exact format.

## Details

- The main page uses a centered layout with the composer at the top and the feedback list below.
- Composer behavior:
  - User types free-form feedback text.
  - On submit, the frontend calls a backend endpoint that triggers the LLM draft flow (see US-03).
  - The LLM response is rendered inline beneath the input.
- The LLM response area shows:
  - An editable draft ticket (title + summary, and structured fields for bugs).
  - An optional single follow-up question (from US-03) inline.
  - Duplicate suggestions list (from US-04).
- Controls:
  - Button to ask the LLM for a draft (if distinct from final submit).
  - Edit controls for the generated draft (title and description fields at minimum).
  - A primary Submit button that creates the final feedback item.

## Acceptance Criteria

- The main page shows a centered AI-style composer above the feedback list.
- Users can type arbitrary feedback text into the composer.
- Triggering the draft action calls the backend and renders:
  - An editable draft ticket.
  - At most one follow-up question (if present).
  - A duplicate suggestions section (empty state if none).
- The user can edit the draft fields before final submission.
- Pressing the final Submit button sends the structured draft to the backend to create the feedback item (see FEAT-01).
